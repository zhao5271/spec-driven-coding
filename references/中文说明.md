# Spec 驱动编码中文说明

这份文档给人工快速阅读。真正的触发和执行规则以 `SKILL.md` 为准。

## 这套 Skill 做什么

它把规划从聊天里拉回文件里，核心文件是：

- `change.toml`
- `spec.md`
- `tasks.md`
- `log.md`

适合这些场景：

- 给仓库初始化 `code_copilot/`
- 新需求先写规格再动代码
- bug 修复先建 change package
- 多层改动需要统一规划和恢复上下文

## 最短流程

### 1. 初始化仓库

```bash
python3 scripts/scaffold_package.py --target /path/to/repo --project-name my-app
```

### 2. 创建变更包

```bash
python3 scripts/create_change.py --target /path/to/repo --name add-bulk-import --title "Add bulk import flow"
```

会生成：

- `change.toml`
- `spec.md`
- `tasks.md`
- `log.md`
- `decisions.md`
- `review.md`

### 3. 先写再做

先补 `spec.md`，再补 `tasks.md`。

任务至少写清：

- `Task ID`
- `Goal`
- `Files`
- `Verification`
- `Expected outcome`
- `Depends on`

`Depends on` 不是备注，而是执行闸门。

### 4. 审批并开做

```bash
python3 scripts/approve_change.py --target /path/to/repo --change add-bulk-import
python3 scripts/update_change_status.py --target /path/to/repo --change add-bulk-import --status in_progress --current-task T1
```

需要补任务字段时，用：

```bash
python3 scripts/update_task_fields.py --target /path/to/repo --change add-bulk-import --task-id T1 --goal "增加请求校验"
```

### 5. 校验和收尾

```bash
python3 scripts/update_change_status.py --target /path/to/repo --change add-bulk-import --status verifying
python3 scripts/validate_change.py --target /path/to/repo --change add-bulk-import
python3 scripts/close_change.py --target /path/to/repo --change add-bulk-import
```

## 记忆口令

- `建包`
- `写规`
- `开做`
- `续做`
- `校验`
- `收尾`

含义：

- `建包`：创建 change package
- `写规`：写或改 `spec.md`
- `开做`：进入执行阶段
- `续做`：用 `change_catchup.py` 恢复上下文
- `校验`：跑 `validate_change.py`
- `收尾`：跑 `close_change.py`

## 关键规则

- No Spec, No Code
- Spec is Truth
- Reverse Sync
- `tasks.md` 是执行快照
- `log.md` 是会话日志

如果代码已经改了，但 `tasks.md / log.md` 还没跟上，先补文档，再继续。
