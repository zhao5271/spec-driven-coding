# Task Splitting Examples

Use these examples when the user asks for concrete task decomposition rather than only a high-level spec.

## API Endpoint

- Task 1: request schema and boundary validation
- Task 2: service logic and persistence changes
- Task 3: client mapping and result rendering

## UI Flow

- Task 1: page flow and state model
- Task 2: API calls, loading, and error states
- Task 3: automated or scripted critical-path checks

## Background Job

- Task 1: job creation entry point and payload validation
- Task 2: worker execution, retries, and status updates
- Task 3: progress visibility or completion flow

## Data Migration

- Task 1: migration or backfill script
- Task 2: read and write path compatibility
- Task 3: rollout checks, observability, and cleanup
